向量整数

这篇文章要说的是如何将向量转成整数，从而非常简单的并行完成向量各维度的线性运算（加、减、数乘）并压缩向量所占空间。对小范围低维向量的线性运算和压缩存储上可以产生很大提升。不是小范围但低维，或者高维但是小范围也可以。高维大范围向量可以做的部分的并行。

矩阵与整数

矩阵的整数表示法

图可以用矩阵来表示，如图

但是但矩阵很稀疏时，空间浪费会很大。

所以想节省一下空间。

要表达一个矩阵只需要记录下每一个 $ (x_i,y_i) -> w_i $

可以构造 $ posW_i = w_isize^2+x_isize+y_i $ 来代表 $ (x_i,y_i) -> w_i $

$w_i = posW_i / size^2 $

$ y_i = posW_i%size$

$ x_i = posW_i%size^2 -y_i$

$ posW_i $与$ (x_i,y_i) -> w_i $相互转化的运算复杂度都是常数。

这样就可以用 vector<int> posWset; 来表示一个矩阵。假设$ n*n$的矩阵上只有k个值,那么占用空间会下降很多很多。

向量与整数

进一步这个方法可以表示一个有限大的向量
$ \vec v (x,y,z)$
也可以等价于一个数
$V=xsize^2 + ysize + z $

向量整数的加减乘性质

这个表示有一些比较好的性质。

向量整数的加法性质

设
$V_1=x_1size^2 + y_1size + z_1 \$
$V_2=x_2size^2 + y_2size + z_2 \$
$V_1+V_2=(x_1+x_2)*size^2 + (y_1+y_2)*size + (z_1+z_2) \$

等价于$ \vec v_1 + \vec v_2$ ，用这种数值加法的好处是可以将一次向量加法简化成一次整数加法，对于三维向量在多次进行加法的情况下可以节省三分之二的空间，少做三分之二次加法。

加法到减法的扩展

因为V的表示范围是有限大的第一象限，如果要进行减法，可能会出现负数超出表示范围，对此我们可以通过对x,y,z加一个基数来解决这个问题。这个基数是约定的，不占空间，也不影响时间复杂度。

我自己感觉这个加减法挺有意义的。向量->很多次整数运算->向量 比 向量->很多次向量运算->向量 要划算很多。

整数表示法的向量内积

$ V_1V_2 = x_1x_2size^2+(x_1y_2+x_2y_1)size+y_1y_2$不能通过一步运算得到$ \vec v_1 \vec v_2$。原方法需要两次乘法一次加法，三步。这种方法也需要三步，一步乘法，一步除法，一步取余。但是，但当$ size = 2^n$时除法和取余会退化成右移和与两个位操作。两个位操作应该时间比一次乘法和一次加法用的时间短很多。

整数表示法的向量数乘

这个很容易看出直接乘上这个数就行。显然对于3维向量，本来要进行三次乘法，现在只需要一次乘法。

更灵活的向量整数

其实这种表示图、矩阵和向量的方法把size = m*n也是可以的，会变得更加灵活，更能充分利用空间。

对于$mn$的矩阵，$pos = w(mn)^2+x*mn + y$ 向量也是同样的道理。

向量整数优势总结

  1. 向量的线性运算速度优于传统方式。n维向量间的加减数乘操作只需要一步运算就并行计算了所有维度，显著提升了速度。不过只是在计算时可以并行，输出时还要串行计算要输出的值。
  2. 向量的存储效率优于传统方式。n维向量转成一个数值，显著提高了空间利用率。
  3. 是稀疏矩阵的空间复杂度最低的数据结构。
  4. 矩阵乘法时间复杂度感觉上不是很高。这方面我没细追究。

点与整数

显然，一个向量也可以看作一个点。但是单独将点拿出来是有意义的。和图形学的点和向量一样有如下操作。

定义向量 = { int pos, char m == 0}; 简记v

定义点 = { int pos, char m !=0}; 简记p 初始时 m = 1

p1+p2 = p3 p1和p2的中点

p2 - p1 = v1 p1到p2的向量

p1 + v1 = p2 p1进行v1的位移

v1 + v2 = v3 向量的加法

v2 - v1 = v3 向量的减法

点输出或某些运算时 pos = pos / m

可见整数化后的点和向量进行加减依旧着很好的运算实际意义。

空间分析

最后，现在的计算机都是64位的，处理数据的字长是64.

对于上面的pos，$pos<(mn)^3. $

$1000000^3 =1000000000000000000<2^{64}=18446744073709551616$

前者比后者少一位，差了18倍以上

所以用整数表示向量或矩阵在空间上一般是十分充分的，不太可能不够用。

如果换成32位整数，也有$100010001000<2^{32} $

假设$size = 2^m$

对于n维向量来说，它的pos小于$(2^m)^n$ ，一个int状态数有$ 2^{32}$ 种。要存下来空间需要满足

$ 2^{mn}<(2^{32})^k$ ，k就是所需int的个数。两边取$log_2$ 可得 mn < 32k

所有向量整数所需空间为$\frac {mn}{32}$ 个int ，压缩率是$ \frac {m}{32}$，并产生了并行性。

如果用int64做单位，所需空间为$\frac {mn}{64}$ 个int ，压缩率是$ \frac {m}{64}$，同样有并行性。

当m=10 int64为一储存单位 可以存储

对于m这个参数，由之前“更灵活的向量整数”部分可知 $ m_i$可以按实际需要在不同维度上的改变，使得n个$ m_i$的乘积在满足条件的前提下更小，提高空间压缩率。m越小可以并行计算的也越多。在m=10时，int64不做任何调整就可以直接并行计算6维向量的线性运算。反之如果一般常用的是低维（三维或者两维），那么m可以开的更大大点。

对于一个特别高维的向量我们可以用一个向量整数的向量来表达，但并行率和压缩率还保持在原有水平。
